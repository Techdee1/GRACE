# NFIU STR Submission Guide (Operational Summary)

Last Updated: 2026-04-24
Scope: Internal analyst reference for GRACE agent-assisted STR drafting.

## Purpose
This guide defines what analysts must verify before filing any STR draft generated by the GRACE agent.

## Non-Negotiable Rule
Agent output is a draft only and must remain in PENDING_REVIEW until human compliance approval.

## Minimum STR Checklist
1. Confirm case reference and reporting period are accurate.
2. Confirm primary suspicious pattern is clearly described.
3. Confirm affected entities are correctly identified.
4. Confirm risk rationale references evidence, not assumptions.
5. Confirm recommendation (ESCALATE/MONITOR/INSUFFICIENT_EVIDENCE) is justified.
6. Confirm red flags are specific and traceable to transaction records.
7. Confirm no unmasked sensitive identifiers appear in narrative text.

## Required Evidence Attachments
1. Candidate summary with candidate IDs.
2. Entity risk score table.
3. Extract of relevant transaction rows.
4. Analyst review notes and sign-off metadata.

## Filing Readiness Criteria
A draft is filing-ready only when:
1. Status has been changed from PENDING_REVIEW by authorized staff.
2. Compliance reviewer confirms evidence sufficiency.
3. Legal/regulatory wording has been reviewed.

## Rejection Conditions
Reject and return for rework if:
1. Risk conclusions cannot be mapped to candidate evidence.
2. Narrative includes speculative statements without support.
3. Data quality issues remain unresolved.
4. Missing mandatory sections or metadata.
